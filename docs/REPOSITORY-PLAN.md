# Repository Plan

## Primary repository: `ai-static-site-preflight`

Purpose: provide local-first checks for AI-generated static HTML before a page
is published.

Public scope:

- Browser-based HTML checker
- Technical validation rules
- Tests, issue templates, and contributor documentation
- Future CLI and GitHub Action

Private scope:

- Marketing strategy
- Conversion heuristics
- Sales-copy prompts
- Client interview materials
- Service-delivery templates

## Optional companion repository: `ai-lp-github-pages-starter`

Purpose: provide a minimal, generic example of publishing an AI-generated static
page through GitHub Pages.

Build this after the checker has an initial release. Keep it limited to
deployment steps, replaceable sample files, and a technical publishing
checklist.

## Optional companion repository: `static-business-page-template`

Purpose: provide a simple static profile and service-information page for small
businesses and independent professionals.

Build this only if a separate template is useful after the primary checker and
publishing starter have real usage. Keep it generic and avoid prescriptive
sales-funnel design.
