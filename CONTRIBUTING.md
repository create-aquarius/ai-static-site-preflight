# Contributing

Thank you for helping improve AI Static Site Preflight.

## Good first contributions

- Report a generated-HTML issue that the current rule set misses.
- Improve an existing rule message.
- Add a focused test case.
- Improve documentation for GitHub Pages users.
- Propose an accessibility or static-hosting compatibility check.

## Before opening a pull request

1. Keep the change focused on technical HTML quality, accessibility, or static
   publishing compatibility.
2. Add or update tests when changing `checker.js`.
3. Run `npm test`.
4. Avoid adding remote tracking, remote HTML analysis, or third-party runtime
   dependencies without discussing the proposal in an issue first.

## Scope boundary

This project does not include marketing scoring, sales-copy generation, or
conversion optimization advice. Rule proposals should be explainable as
technical checks for a static website.
