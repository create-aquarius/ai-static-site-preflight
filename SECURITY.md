# Security Policy

## Supported versions

Security fixes are applied to the latest release.

## Reporting a vulnerability

Please do not include sensitive HTML, credentials, API keys, customer data, or
private website content in a public issue.

Until a private reporting channel is added, open a public issue with a minimal
description that contains no sensitive details and ask for a private follow-up
method.

## Local-first design

The browser version performs checks locally. It should not upload, store, or
send the HTML being checked. Pull requests that add remote processing, tracking,
or analytics must be discussed before implementation.
